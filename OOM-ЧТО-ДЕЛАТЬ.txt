CURSOR ПАДАЕТ С OOM (The window terminated unexpectedly) — ЧТО ДЕЛАТЬ

1. При появлении окна с ошибкой:
   ✓ Обязательно поставьте галочку «Don't restore editors».
   ✓ Нажмите Reopen (не Close).
   Иначе откроются все старые вкладки и память снова кончится.

2. Самое эффективное — открывать НЕ весь проект, а одну папку:
   • Дважды щёлкните по файлу Open-Cursor-Web.bat (только фронт) или Open-Cursor-Backend.bat (только API) в корне проекта — откроется Cursor с одной папкой, без крашей.
   • Или вручную: File → Open Folder → выберите папку web или backend.
   Так Cursor индексирует в разы меньше файлов — OOM исчезает или бывает реже.

3. После открытия:
   • Закройте лишние вкладки (оставьте 3–5).
   • Не открывайте сразу несколько тяжёлых файлов (Dashboard.tsx, OrdersMap.tsx).

4. Подробно: см. файл CURSOR-TIPS.md в корне и docs/КАК-УЛУЧШИТЬ-ПАМЯТЬ-CURSOR.md
